# Отчет: Черно-белый шейдер квадратами при изменении резолюции воксель ремеша

## 📋 Обзор

Система визуализации вокселей в SculptSP представляет собой **предпросмотр сетки воксельного разрешения в реальном времени**. Когда пользователь меняет параметры ремеша, на поверхность меша накладывается черно-белый паттерн квадратов, который визуально показывает размер воксельной ячейки, которая будет использована при ремешировании.

---

## 🔧 Основные файлы реализации

### 1. **ShaderVoxelChecker.js** (`src/render/shaders/`)

**Назначение:** Основной GLSL шейдер, отвечающий за отрисовку черно-белого паттерна квадратов.

**Ключевые компоненты:**
- **Vertex Shader:** Трансформирует вершины в view space (пространство камеры)
- **Fragment Shader:** Создает паттерн на основе позиции в пространстве
  - Делит позицию на `uStep` (размер ячейки воксела) 
  - Использует `modulo` для определения четности строки/столбца
  - Рендерит серый цвет (0.5) с прозрачностью 0.15 для одного паттерна
  - Рендерит черный цвет (0.0) с прозрачностью 0.6 для другого паттерна

**Uniforms:**
- `uMVP` — матрица Model-View-Projection
- `uMV` — матрица Model-View
- `uStep` — размер ячейки воксела (основной параметр, управляющий размером квадратов)

**Attributes:**
- `aVertex` — позиции вершин 3D меша (vec3)

---

### 2. **Remesh.js** (`src/editing/`)

**Назначение:** Основной алгоритм воксельного ремеширования, управляет разрешением и вычисляет размер шага.

**Ключевые свойства:**
- `Remesh.RESOLUTION` — разрешение воксельной сетки (по умолчанию 150)
  - Чем выше значение, тем меньше воксели
  - Чем ниже значение, тем больше воксели
- `Remesh.BLOCK` — флаг для блочного стиля ремеширования
- `Remesh.SMOOTHING` — флаг для тангенциального сглаживания

**Ключевые функции:**
- `computeVoxelStep(meshes)` — **Критическая функция**
  - Вычисляет размер шага воксела: `step = maxDimension / Remesh.RESOLUTION`
  - `maxDimension` — максимальное измерение bounding box всех мешей
  - Результат используется для передачи в шейдер как `uStep`

- `createVoxelData()` — Создает структуру 3D сетки воксельных ячеек
- `voxelize()` — Заполняет воксельную сетку поле расстояния (distance field)
- `floodFill()` — Определяет внутренние и внешние воксели
- `remeshAsync()` — Асинхронное ремеширование с коллбеками прогресса

---

### 3. **GuiTopology.js** (`src/gui/`)

**Назначение:** UI контроли для управления параметрами ремеша и триггеры предпросмотра.

**Ключевые методы:**
- `remeshResolution(val)` — Устанавливает `Remesh.RESOLUTION` и обновляет предпросмотр
  ```javascript
  Remesh.RESOLUTION = val;
  this._updateVoxelSizeLabel();
  ```

- `_updateVoxelSizeLabel()` — Вычисляет размер воксела и вызывает предпросмотр
  - Вызывает `Remesh.computeVoxelStep(meshes)` 
  - Затем вызывает `Scene.updateVoxelPreview(step, meshes)`

**Триггеры предпросмотра:**
- Движение мыши по слайдеру разрешения (hover)
- Фокус на текстовое поле ввода разрешения
- События слайдера/поинтера
- Нажатие клавиши `-X` (модальное окно выбора разрешения)
- Нажатие клавиши `Ctrl+X` для применения ремеша

---

### 4. **Scene.js** (`src/`)

**Назначение:** Основной рендер-движок сцены, отвечает за отрисовку предпросмотра воксельной сетки.

**Ключевые методы:**

- `updateVoxelPreview(step, meshes)` — Сохраняет данные для предпросмотра
  - `step` — размер ячейки воксела (от `Remesh.computeVoxelStep()`)
  - `meshes` — массив мешей для отрисовки паттерна
  - Сохраняет в `this._voxelPreview = {step, meshes}`

- `_renderVoxelPreview()` — Основная функция отрисовки
  - Проверяет наличие `_voxelPreview`
  - Для каждого меша вызывает: `shader.draw(mesh, step)`
  - Использует `ShaderVoxelChecker`

**Параметры отрисовки:**
- Рендерится в **opaque pass** (непрозрачный проход, строка 324)
- Позиция: после фона, перед прозрачными объектами
- Blend mode: `SRC_ALPHA, ONE_MINUS_SRC_ALPHA`
- Polygon offset для предотвращения z-fighting (наложения артефактов)

**Хранилище:**
- `this._voxelPreview` — объект вида `{step: number, meshes: Mesh[]}`

---

### 5. **SurfaceNets.js** (`src/editing/`)

**Назначение:** Алгоритм Surface Nets для реконструкции поверхности из воксельной сетки.

**Использование:**
- Преобразует 3D сетку воксельных ячеек в полигональный меш
- Альтернатива Marching Cubes
- `SurfaceNets.BLOCK` — контролирует стиль выходной геометрии

---

### 6. **MarchingCubes.js** (`src/editing/`)

**Назначение:** Классический алгоритм Marching Cubes для реконструкции поверхности.

**Использование:**
- Альтернативный метод преобразования воксельной сетки в полигоны
- `MarchingCubes.BLOCK` — контролирует стиль выходной геометрии

## 🎯 Формула вычисления размера воксела

```
step = maxDimension / Remesh.RESOLUTION

Где:
  step — размер одной воксельной ячейки в мировом пространстве
  maxDimension — максимальное измерение bounding box меша
                 (max(width, height, depth))
  RESOLUTION — число воксельных ячеек по максимальному измерению
               (по умолчанию 150)

Примеры:
  Если maxDimension = 150, RESOLUTION = 150
  → step = 1.0 (очень мелкие воксели)
  
  Если maxDimension = 150, RESOLUTION = 50
  → step = 3.0 (крупные воксели)
  
  Если maxDimension = 300, RESOLUTION = 100
  → step = 3.0 (средние воксели)
```

## 🖼️ Визуальная логика шейдера

```glsl
// Fragment Shader упрощенная логика:

vec3 viewPos = (uMV * vec4(vertexPos, 1.0)).xyz;

// Делим на step для получения координат ячейки
vec3 cellCoord = floor(viewPos / uStep);

// Определяем четность
float isOdd = mod(cellCoord.x + cellCoord.y + cellCoord.z, 2.0);

if (isOdd < 0.5) {
    // Серый цвет с низкой прозрачностью
    gl_FragColor = vec4(0.5, 0.5, 0.5, 0.15);
} else {
    // Черный цвет с высокой прозрачностью
    gl_FragColor = vec4(0.0, 0.0, 0.0, 0.6);
}
```

## 🔄 Жизненный цикл предпросмотра

1. **Инициация:** Пользователь наводит мышь на слайдер разрешения или вводит значение
2. **Обновление разрешения:** `Remesh.RESOLUTION` изменяется на новое значение
3. **Вычисление шага:** `computeVoxelStep()` рассчитывает новый размер воксела
4. **Сохранение в сцену:** `Scene.updateVoxelPreview()` запоминает step и меши
5. **Отрисовка:** На следующем frame `_renderVoxelPreview()` вызывает шейдер
6. **Визуализация:** На экране появляется/обновляется черно-белый паттерн
7. **Применение:** Если пользователь нажимает Ctrl+X, ремеш выполняется с текущим RESOLUTION

## ⚙️ Интеграция компонентов

```
GuiTopology (UI контроллер)
    ├─ Слушает события мыши и клавиатуры
    ├─ Изменяет Remesh.RESOLUTION
    └─ Вызывает Scene.updateVoxelPreview()
         │
         └─ Scene (рендер контроллер)
             ├─ Сохраняет _voxelPreview данные
             ├─ На каждом frame вызывает _renderVoxelPreview()
             └─ ShaderVoxelChecker
                 ├─ Получает mesh и step
                 ├─ Вычисляет паттерн в fragment shader
                 └─ Рендерит квадраты на меш
```

## 📝 Заключение

Черно-белый шейдер с квадратами — это **реал-тайм预览 система**, которая помогает пользователю понять, как будет выглядеть ремеш при текущих настройках. Система состоит из:

1. **Шейдера** (ShaderVoxelChecker) — отвечает за визуальное представление
2. **Вычислителя** (Remesh.computeVoxelStep) — преобразует RESOLUTION в геометрический размер
3. **UI контроллера** (GuiTopology) — управляет взаимодействием пользователя
4. **Рендер-системы** (Scene) — координирует отрисовку

Все компоненты тесно интегрированы для обеспечения плавного и интуитивного пользовательского опыта при настройке параметров воксельного ремеша.
